# 🔐 [STEP 6] 소셜 로그인 & 인프라 세팅 통합 가이드

이 가이드는 **Supabase Auth**를 통해 구글/깃허브 로그인을 구현하고, 이를 **Render** 서버와 연동하기 위한 전체 프로세스를 다룹니다.

## 1단계: Supabase 프로젝트 설정 (인증 센터)

가장 먼저 우리 앱의 인증을 담당할 Supabase 프로젝트의 기본 정보를 확보해야 합니다.

1. **API 정보 확보**: Supabase Dashboard -> **Settings (톱니바퀴)** -> **API** 메뉴 이동.
    
    - **Project URL**: `https://xyz.supabase.co` 형태의 주소 (클라이언트/서버 공통)
        
    - **anon/public key**: 클라이언트(`App.jsx`)에서 사용할 공개 키.
        
    - **service_role/secret key**: 서버(`server.js`)에서 토큰 검증 시 사용할 마스터 키. (**절대 외부 노출 금지**)
        
2. **Redirect URL 설정 (가장 중요)**: **Auth** -> **URL Configuration** 메뉴 이동.
    
    - **Site URL**: `https://secret-journal.onrender.com` (본인의 실제 Render 주소)
        
    - **Redirect URLs**: `http://localhost:3000/**` 추가 (로컬 테스트용)
        
    - _이 설정이 없으면 로그인 후 앱으로 돌아올 때 `bad_oauth_state` 에러가 발생합니다._
        

## 2단계: 소셜 로그인 제공자(Provider) 설정

### ① GitHub (OAuth App 등록)

1. **GitHub Settings** -> **Developer settings** -> **OAuth Apps** -> **New OAuth App**.
    
2. **Homepage URL**: 본인의 Render 주소 입력.
    
3. **Authorization callback URL**: Supabase의 **Auth -> Providers -> GitHub** 메뉴에 적힌 주소를 복사해서 입력.
    
4. 발급된 **Client ID**와 **Client Secret**을 Supabase GitHub 설정 창에 입력하고 **Save**.
    

### ② Google (OAuth Client ID 등록)

1. **Google Cloud Console** -> 프로젝트 선택 -> **API 및 서비스** -> **사용자 인증 정보**.
    
2. **사용자 인증 정보 만들기** -> **OAuth 클라이언트 ID**.
    
3. **애플리케이션 유형**: 웹 애플리케이션.
    
4. **승인된 리디렉션 URI**: Supabase의 **Google Provider** 설정에 있는 **Callback URL** 추가.
    
5. 발급된 정보를 Supabase에 등록.
    

## 3단계: 환경 변수(Environment Variables) 세팅

앱이 실행되는 환경(로컬 vs 배포)에 따라 필요한 변수들을 등록해야 합니다.

### ① 로컬 환경 (client/.env 및 server/.env)

- **Client**: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`
    
- **Server**: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `DATABASE_URL`, `ORIGIN_URL`
    

### ② 배포 환경 (Render Dashboard)

Render 관리 화면의 **Environment Variables** 탭에서 아래 값들을 각각 입력하세요.

**[Web Service - Server]**

- `SUPABASE_URL`: Supabase 프로젝트 URL
    
- `SUPABASE_SERVICE_ROLE_KEY`: 서비스 롤 비밀 키
    
- `DATABASE_URL`: Supabase Database URI (포트 6543)
    
- `ORIGIN_URL`: 실제 리액트 배포 주소 (`https://...`)
    

**[Static Site - Client]**

- `VITE_SUPABASE_URL`: Supabase 프로젝트 URL
    
- `VITE_SUPABASE_ANON_KEY`: 아논 공개 키
    
- `VITE_API_URL`: 실제 서버 배포 주소 (`https://...-api.onrender.com`)
    

## 4단계: 코드 통합 및 검증

1. **Frontend (`App.jsx`)**: `createClient` 호출 시 하드코딩된 주소가 아닌 환경 변수를 사용하도록 수정합니다.
    
2. **Backend (`server.js`)**: `express-session` 관련 코드를 모두 제거하고, `checkAuth` 미들웨어가 `supabaseAdmin.auth.getUser(token)`를 호출하여 JWT를 검증하도록 수정합니다.
    
3. **DB Schema**: `notes` 테이블에 `user_id` 컬럼이 `UUID` 형식을 지원하는지 확인합니다 (Supabase Auth 유저 ID는 UUID입니다).
